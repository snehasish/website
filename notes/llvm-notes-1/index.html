Fri 21 August 2015

Introduction

These notes are based on the configuration of a LLVM based project which uses the autoconf build system. LLVM is configured with --enable-optimized --enable-assertions --enable-targets=x86,cpp and built in a separate build directory.

Boost and LLVM

Debugging Projects

If you have your project in --disable-optimized mode then it will look for the Debug+Asserts directory in the LLVM build folder (specified during configure). However it is possible to link against the Release+Assert libs while your project is configured for Debug+Asserts by adding a symlink in the LLVM build folder (and it works, though you only get symbols for your project): 

$ ln -s Release+Asserts Debug+Asserts
